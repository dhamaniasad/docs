Smart Memory is a revolutionary feature within MemoryPlugin that makes it easier and more efficient to work with large numbers of memories.

## The Problem Smart Memory Solves

Until recently, MemoryPlugin operated as a simple list. You would add new memories to the list, and the entire list would be shared with the AI when required. While this is useful, it has significant limitations:

<CardGroup cols={3}>
  <Card title="Irrelevant Information" icon="filter">
    Dumping the entire memory list feeds tons of irrelevant information to the AI, potentially reducing performance by confusing the model
  </Card>
  <Card title="Context Window Bloat" icon="window-maximize">
    Not every conversation needs full details of every memory. As you add more memories, they take up more space, leaving less room for your actual work
  </Card>
  <Card title="Higher Costs" icon="dollar-sign">
    All API-based tools charge by the token, so more information in the context window means higher costs and faster rate limiting
  </Card>
</CardGroup>

## What is Smart Memory?

Smart Memory is an intelligent system that creates efficient, AI-optimized summaries of your memory collections.

### How It Works

Smart Memory operates at the [Memory Bucket](/features/memory-buckets) level:

1. **Analysis**: Takes all memories within a given bucket
2. **Summarization**: Creates a high-level overview of the memories
3. **Contextualization**: Provides the AI with summary information instead of full memory text
4. **On-Demand Access**: Gives the AI details on what additional information is available and allows it to request full memories when necessary

<Info>
The AI receives a concise overview but can always request the full memory details when they're actually needed for the conversation.
</Info>

## Benefits

### Token Savings
Smart Memory can save up to **90% on tokens** by:
- Providing summaries instead of full memory text
- Only loading detailed memories when actually needed
- Eliminating redundant or irrelevant information from context

### Improved Performance
- **Reduced confusion**: AI receives only relevant, summarized information
- **Faster processing**: Less text to process means faster responses
- **Better focus**: AI can concentrate on what's actually important for the conversation

### Enhanced Scalability
- **Handle more memories**: Your memory collection can grow without performance degradation
- **Maintain responsiveness**: Large memory sets don't slow down your AI interactions
- **Preserve context space**: More room for your actual work and conversation

## Technical Implementation

### Memory Hierarchy
Smart Memory creates a hierarchical structure:

```
Bucket: "Work Project Alpha"
├── Summary: "Project involving client onboarding system..."
├── Key Topics: ["authentication", "user interface", "database design"]
├── Available Details: [Memory IDs and brief descriptions]
└── Full Memories: [Available on request]
```

### AI Integration
When Smart Memory is active:

1. **Initial Load**: AI receives the bucket summary and available memory topics
2. **Contextual Awareness**: AI understands what detailed information is available
3. **Selective Access**: AI can request specific memories when they become relevant
4. **Dynamic Loading**: Full memory content is loaded only when needed

## Compatibility

Smart Memory works with all MemoryPlugin integrations:

- **Browser Extension**: Automatically uses Smart Memory when available
- **MCP Server**: Provides Smart Memory summaries through MCP protocol
- **Custom GPT**: Integrates Smart Memory into GPT function calls
- **API**: Smart Memory accessible through API endpoints
