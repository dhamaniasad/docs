## Why Claude + MemoryPlugin?

Claude is special because:
- **Native MCP Support**: Anthropic created the MCP protocol
- **Seamless Integration**: Built-in support in Claude Desktop
- **Automatic Loading**: Memories load without manual intervention
- **Better Performance**: Direct protocol integration

## Quick Start Guide

### Method 1: MCP Server (Claude Desktop)

<Steps>
  <Step title="Install Claude Desktop">
    Download from [claude.ai/download](https://claude.ai/download)
  </Step>
  <Step title="Install Node.js">
    Required for MCP server - get from [nodejs.org](https://nodejs.org)
  </Step>
  <Step title="Configure MCP">
    Add MemoryPlugin to your Claude MCP configuration:
    ```json
    {
      "mcpServers": {
        "memoryplugin": {
          "command": "npx",
          "args": ["@memoryplugin/mcp-server"],
          "env": {
            "MEMORY_PLUGIN_TOKEN": "your-token-here"
          }
        }
      }
    }
    ```
  </Step>
  <Step title="Restart Claude">
    Restart Claude Desktop to load the MCP server
  </Step>
  <Step title="Use Memories">
    Your memories are now available in all Claude conversations
  </Step>
</Steps>

### Method 2: Browser Extension (Claude.ai)

<Steps>
  <Step title="Install Extension">
    Add the [MemoryPlugin Browser Extension](/integrations/browser-extension)
  </Step>
  <Step title="Visit Claude.ai">
    Go to [claude.ai](https://claude.ai) in your browser
  </Step>
  <Step title="Activate MemoryPlugin">
    Click the MemoryPlugin button to inject memories
  </Step>
  <Step title="Start Chatting">
    Claude now has access to your memory context
  </Step>
</Steps>

## MCP Server Configuration

### Basic Setup

Location of Claude's configuration file:
- **macOS**: `~/Library/Application Support/Claude/config.json`
- **Windows**: `%APPDATA%\Claude\config.json`
- **Linux**: `~/.config/Claude/config.json`

### Advanced Configuration

```json
{
  "mcpServers": {
    "memoryplugin": {
      "command": "npx",
      "args": ["@memoryplugin/mcp-server"],
      "env": {
        "MEMORY_PLUGIN_TOKEN": "your-token-here",
        "DEFAULT_BUCKET": "Work",
        "AUTO_LOAD": "true"
      }
    }
  }
}
```

### Multiple MCP Servers

Claude supports multiple MCP servers simultaneously:

```json
{
  "mcpServers": {
    "memoryplugin": {
      // MemoryPlugin configuration
    },
    "filesystem": {
      // Another MCP server
    },
    "github": {
      // Yet another MCP server
    }
  }
}
```

## Using MemoryPlugin with Claude

### Memory Operations

In Claude Desktop with MCP:
- **Load memories**: Happens automatically or with "Load my memories"
- **Create memory**: "Remember that..." or "Add to memory..."
- **Search memories**: "What do you remember about..."
- **Switch buckets**: "Load memories from my Work bucket"

### Best Practices

<AccordionGroup>
  <Accordion title="Project Organization">

    - Store project-specific context as memories
    - Keep general memories in the default bucket

  <Accordion title="Conversation Flow">

    - Explicitly request specific buckets when needed

    - Review and clean memories periodically
  </Accordion>
  <Accordion title="Performance Optimization">
    - Use Smart Memory for large collections
    - Keep individual memories concise
    - Archive old project memories
    - Use bucket organization to reduce noise
  </Accordion>
</AccordionGroup>

## Troubleshooting Claude Integration

<AccordionGroup>
  <Accordion title="MCP server not connecting">
    - Check Node.js is installed: `node --version`
    - Verify config file location and syntax
    - Ensure auth token is valid
    - Restart Claude Desktop completely
    - Check logs in Claude's developer console
  </Accordion>
  <Accordion title="Memories not loading">
    - Confirm MCP server is running (check Claude's UI)
    - Try manually requesting: "Load my memories"
    - Verify bucket selection is correct
    - Check MemoryPlugin Dashboard for memories
  </Accordion>
  <Accordion title="Browser extension issues">
    - Refresh claude.ai page
    - Check extension is enabled for claude.ai
    - Clear site cookies and cache
    - Try incognito mode to isolate issues
  </Accordion>
  <Accordion title="Performance problems">
    - Reduce memory count with Smart Memory
    - Organize into more specific buckets
    - Archive old, unused memories
    - Check system resources (CPU/RAM)
  </Accordion>
</AccordionGroup>
